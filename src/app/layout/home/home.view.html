My this test work was inspired by next articles:

1. "Angular 1 Style Guide" by John Papa
2. Angular 1.5 - ES6/ES2015 Style Guide
3. Build an Angular 1.5 application with ES6 and Components

Of course, the most interesting part in this work is my own vision, how to create common structure, which will be convenient in the subsequent scaling of the application. Components are the heart of application, and I tried to separate them onto 2 types: 1) "layout-components" (which are used as a targets in routing) and should be used like containers for other components 2) "ui-components", which should be looks like an atomic structures of interface with encapsulation of its own behaviours and API, which should be robust and minimalistic in ideal case.
